XY Pair

An XY Pair is a data format existing in many Dusk methods. It can be provided in one of three forms. If a method is specified as using X and Y, being of type XY Pair-X and XY Pair-Y, respectively, this is what it refers to. For a method that uses an XY Pair, any of these forms may be used.

Form 1

The most basic form of an XY Pair is provided in the form of two number variables, X and Y.

local x, y = 10, 10
local tile = map.layer[1].tile(x, y)
local tile2 = map.layer[1].tile(11, 10)

Form 2

The second form of an XY Pair is provided from a table with the keys x and y. These values are then used as the required X and Y.

local xy = {
	x = 10,
	y = 10
}

local tile = map.layer[1].tile(xy)
local tile2 = map.layer[1].tile({x = 11, y = 10})

Why is this useful? Any table with X and Y values in it may be used as the parameter, without you having to "unpack" it and provide t.x and t.y.

local function onTouch(event)
	local tileX, tileY = map.layer[1].pixelsToTiles(event)
end

Form 3

The third form of an XY Pair is also provided from a table, but instead of x and y keys, the table has [1] and [2] indices. This form is evaluated last; if a table has x and y keys, they will be used.

local xy = {10, 10}
local tile = map.layer[1].tile(xy)
local tile2 = map.layer[1].tile({11, 10})
